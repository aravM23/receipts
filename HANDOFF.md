# STANLEY — Receipt Cards · Agent Handoff

Read this first. It's the context a fresh agent (or a fresh laptop) needs to
keep working on this project without re-discovering everything.

---

## 1. What this is

An in-person **bar activation**: a guest types their Instagram handle, and the
app prints them a **receipt** with:

- A profile block (photo, handle, posts / followers / following, verified badge)
- **3 personal insights** ("Stanley says…")
- A **Creator archetype** (e.g. "Story Collector")
- A **Creator cocktail** recommendation
- A **QR code** that opens their virtual card at `/c/[slug]`

Data comes only from **HikerAPI** (public IG scraping, no OAuth). Copy is
generated by an LLM, grounded so it only cites **real numbers** from the data.
The on-screen card and the printed thermal receipt share one component.

The receipt is the primary deliverable. `/preview` renders the design with
mock data and **needs no API keys** — use it for design work.

## 5. Run it locally

```bash
npm install
cp .env.example .env.local     # then fill secrets (see §6)
npm run dev                    # default port 3000
```

- Design check (no keys): `/preview`
- Real card: type a public handle on `/`
- Without `OPENAI_API_KEY` → falls back to rule-based copy. Without a Hiker key
  → generation returns 503.

**Take a screenshot of the receipt** (used heavily during design work):

```bash
VW=480 VH=1400 node scripts/shot.mjs http://localhost:3000/preview /tmp/out.png
```

## 7. Persistence

`src/lib/store.ts` auto-selects the backend:

- **Upstash Redis** when the Upstash/KV env vars are present (production on
  Vercel — required, since Vercel's filesystem is ephemeral).
- **JSON file** at `./data/store.json` otherwise (local dev).

All store functions are `async`. It holds cards, a monotonic ticket counter,
recent-by-handle dedup, and the print-job queue (Redis sorted set for FIFO).

---

## 8. Printing

The iPad/desktop **never talks to the printer**. It enqueues a job via
`POST /api/print`. A **worker** on the machine wired to the thermal printer
drains the queue and emits ESC/POS (native QR via `GS ( k`).

```bash
PRINT_WORKER_TOKEN=dev-print-token npm run print-worker
```

No `PRINTER_DEVICE` → dry-run into `./print-output/*.bin` (+ `.txt` preview).
Real hardware: `PRINTER_DEVICE=/dev/usb/lp0`. Width 42 cols = 80mm (default).
The "Print my receipt" button is silent by design (no browser print dialog).

## 9. Design system

Whole site uses exactly three fonts (loaded in `globals.css`):

- **Plus Jakarta Sans** — UI, labels, headings, the "Stanley" wordmark
- **Inter** — secondary/body
- **Lora** — serif accents (Creator archetype + cocktail names, landing headline)

Legacy CSS vars (`--font-mono/-display/-stanley`) are repointed onto these so
old class names still resolve.

**Receipt is pixel-spec'd** (see `Receipt.tsx`). Current target = **339 × 1288**
(min-height + `justify-content: space-between` opens the vertical rhythm).
Key element sizes: avatar 110, verified badge 22.8, `$` badge 32, QR 128,
archetype box 315 × 161 (breaks ~12px past the dividers each side), header mark
23.2, social icons 21.058 (Instagram +13.19° / LinkedIn -7.163° / X on top,
overlapping). Card background is white; PNG export uses `#ffffff`.

If the client sends more Figma specs, just translate px-for-px into `Receipt.tsx`.

## 11. Gotchas / lessons learned

- **Dev server in the sandbox:** start it as a managed background process
  *without* a trailing `&` (a trailing `&` detaches and it dies). If port 3000
  is taken, pick another (3007/3011 used previously). Confirm with a `curl`.
- **`scripts/shot.mjs`** imports Playwright from the sibling `../cards`
  project's `node_modules` (via an absolute `createRequire` path). If Chromium
  is missing: `node node_modules/playwright/cli.js install chromium` in `cards`.
- **Assets get swapped easily** — verify which PNG is which by opening it, not
  by filename, before wiring it in (we mixed up cocktail vs. archetype box once).
- **No em dashes** in any user-facing copy (and the LLM prompt forbids them);
  `cleanLine` in `generate.ts` strips them as a safety net.
- **LLM grounding:** insights must be anchored to real caption excerpts / real
  numbers from the digest. Keep the "no vague filler" rules in the prompt.
- The earlier `cards/` project is a *separate* app; this one only borrows the
  Playwright binary and some branding ideas.
